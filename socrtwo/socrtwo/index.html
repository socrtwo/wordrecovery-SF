    What this does

    • Parse intact .docx — extracts the document text from word/document.xml and from any embedded headers, footers, footnotes, endnotes, and comments.
    • Repair partially-corrupt .docx — locates the central directory and individual local file headers; recovers entries even if the archive's TOC is damaged.
    • Salvage from raw .doc binaries — scans the OLE compound storage and pulls printable runs of text using both ASCII and UTF-16LE heuristics.
    • RTF — strips control words and unescapes hex/Unicode runs.
    • ODT — same ZIP+XML approach as .docx but on content.xml.
    • Strings scan (last resort) — works on any file, including renamed temp files and recovered fragments from disk.

    All five methods run in parallel and the best result is shown first. Each method's output is also available individually.
